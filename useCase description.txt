print or export build
Предусловие: 
Открыт редактор сборки
Главный поток:
Пользователь нажимает на кнопку, отвечающую за экспорт
Система выводит меню настройки экспорта
Пользователь настраивает экспорт и подтверждает операцию
Система генерирует файл и выводит элемент для скачивания  
Послеусловие: 
Пользователь загрузил файл в выбранном формате

add new component from list
Предусловие:
Открыт редактор сборки. Выбран тип элемента в интерфейсе редактора
Главный поток:
Пользователь нажимает на кнопку, отвечающую за добавление нового элемента сборки
Система выводит список фильтров а так же список предлагаемых компонентов
Пользователь настраивает фильтры и выбирает необходимый компонент
Система добавялет компонент в текущую сборку и отображает в интефейсе 
Послеусловие:
Новый компонент из списка был добавлен в сборку

view build
Предусловие:
У Пользователя есть сборка или extend "search build" дал результат
Главный поток:
Пользователь выбирает сборку и нажимает на кнопку, отвечающую за просмотр выбранной сборки
Система отображает интерфейс просмотра сборки
Если пользователь нажмает на кнопку, отвечающую за переход в режим редактирования то extend "open build editor" (недоступно при расширении из "add new component from build")
Послеусловие:
Открыт интерфейс просмотра выбранной сборки

go to component site
Предусловие:

Главный поток:
Пользователь нажимает на кнопку, отвечающую за открытие списка отслеживаемых компонентов

В новой вкладке открыт сайт компонента

sign in
Предусловие:
У Пользователя есть гугл аккаунт
Главный поток:
Пользователь нажимает на кнопку, отвечающую за вход в аккаунт
Система открывает интерфейс входа в гугл аккаунт
Пользователь авторизируеться
Сервис гугл подтверждает авторизацию
Система фиксирует вход в аккаунт
Послеусловие:
Текущая сесия пользователя становится авторизированной

change users info and permissions
Предусловие:
Пользователь авторизирован как Администратор
Главный поток:
Администратор нажимает на кнопку, отвечающую за управления Пользователями
Система отображает интерфейс изменения информации и прав Пользователей
Администратор изменяет данные о Пользователе и подтверждает изменения
Система сохраняет изменение и вносит их в "cloud users DB" 
Послеусловие:
Информация и права Пользователей изменены и сохранены

update components DB
Предусловие:
Пользователь авторизирован как Администратор
Администратор подготовил данные в необходимом формате или подготовил парсер для нового формата
Главный поток:
Администратор нажимает на кнопку, отвечающую за обновление базы данных компонентов
Система отображает интерфейс для ввода данных необходимых для обновления 
Администратор вводит данные и подтверждает
Система пытается обновить БД и выводит логи в интерфейс обновления
Если произошла ошибка, Система отменяет операцию. Администратор может повторить обновление
Послеусловие:
В Базу данных компонентов добавлены новые компоненты
